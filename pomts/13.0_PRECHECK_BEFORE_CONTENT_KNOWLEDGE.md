# Stage 13.0 — Precheck Before Content, Knowledge and Documents

## Purpose

Audit-only stage before starting Stage 13 Content / Knowledge / Documents.

Goal: determine if the project foundation is ready.

Result must be one of:

- GO
- GO_WITH_CORRECTIONS
- STOP

## Rules

Do not:

- create migrations;
- change database schema;
- install packages;
- modify storefront;
- modify Admin UI;
- implement Stage 13 features.

Only audit and create a report.

## Input

Check:

- Git repository state;
- previous stage reports;
- Medusa modules;
- migrations;
- existing models;
- API routes;
- permissions;
- current content/document systems.

## 1. Git audit

Check:

- current branch;
- latest commit;
- uncommitted changes;
- unfinished migrations;
- temporary files;
- incomplete previous work.

## 2. Stage 12 verification

Find:

`reports/stage-12-report.md`

If missing:

- inspect Git history;
- inspect implementation evidence;
- do not invent completed checks;
- mark confidence level.

Output:

- found/not found;
- evidence;
- confidence.

## 3. Medusa architecture audit

Check:

- backend modules;
- services;
- workflows;
- API routes;
- migrations;
- models.

Create table:

| Area | Exists | Status | Risk |
|---|---|---|---|

## 4. Commerce readiness

Verify:

- products;
- variants;
- cart;
- pricing;
- orders;
- customer/B2B entities;
- RFQ flow.

Status:

READY / PARTIAL / MISSING

## 5. Server domain readiness

Check existence of:

- ServerModel;
- Component;
- ComponentPack;
- ReadyConfiguration;
- StorageTopology;
- Compatibility Rules.

Do not create duplicates.

## 6. Content and documents audit

Find existing:

- articles;
- knowledge base;
- media;
- documents;
- SourceDocument;
- vendor documentation.

Decide:

- extend existing;
- migrate;
- create new entity;
- avoid duplicate.

## 7. Hardware data readiness

Check readiness for:

- HPE;
- Dell;
- Supermicro;
- Huawei.

Check:

- CPU data;
- RAM data;
- RAID data;
- NIC data;
- storage data;
- compatibility data;
- source tracking.

## 8. Import dependency check

Stage 14 dependency:

Check:

- import scripts;
- normalization;
- external IDs;
- source URLs;
- confidence fields.

Do not implement import here.

## 9. Database structure check through code

Without connecting to database check:

- migrations;
- models;
- relations;
- naming conflicts;
- missing indexes.

## 10. API and security check

Verify:

- Admin permissions;
- public/private separation;
- draft protection;
- API ownership rules.

Ensure draft content cannot leak to Store API.

## 11. Conflict detection

Find:

- duplicate entities;
- old experiments;
- conflicting modules;
- inconsistent naming.

Report:

- location;
- risk;
- recommendation.

## 12. Final report

Create:

`reports/stage-13-precheck-report.md`

Report must include:

1. Repository state
2. Stage 12 verification
3. Architecture summary
4. Existing entities
5. Missing dependencies
6. Risks
7. Recommendation
8. Exact next steps

## Final rule

Do not start Stage 13 automatically.

Stop after producing the precheck report and wait for approval.
